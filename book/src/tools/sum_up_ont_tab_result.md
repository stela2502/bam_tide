# sum_up_ont_tab_result

`sum_up_ont_tab_result` summarizes ONT read-tag tables generated during preprocessing workflows.

The tool provides lightweight statistics for:

- observed cell barcode (`raw_cb`) usage
- UMI complexity
- barcode/UMI pair frequencies
- and filtering behavior

It is primarily intended as a quick inspection and QC utility during ONT single-cell workflow development.

---

# Typical Use Cases

- quick QC of ONT preprocessing output
- inspection of barcode complexity
- debugging read-tag generation
- evaluating barcode/UMI filtering thresholds
- validating preprocessing experiments

---

# Basic Usage

```bash
sum_up_ont_tab_result reads.tags.tsv
```

Compressed input is also supported:

```bash
sum_up_ont_tab_result reads.tags.tsv.gz
```

---

# Filtering Parameters

The tool applies simple filtering rules to reduce noise from low-support barcode/UMI combinations.

## Minimum pair count

```bash
--min-pair-count 2
```

Minimum number of observations required for one:

```text
raw_cb + raw_umi
```

combination.

---

## Minimum UMIs per barcode

```bash
--min-cb-umis 3
```

Minimum number of surviving UMIs required for a barcode after pair filtering.

---

# Example

```bash
sum_up_ont_tab_result \
  results/read_tags.tsv.gz \
  --min-pair-count 3 \
  --min-cb-umis 5
```

---

# Input Format

The input is expected to be a TSV file generated by:

- `bam-ont-normalizer`
- or compatible preprocessing workflows

Compressed `.gz` files are supported automatically.
